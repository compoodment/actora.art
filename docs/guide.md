# Guide

Start here if you want to understand the public side of actora.art.

## Reading order

1. [`identity.md`](identity.md)
2. [`codebase.md`](codebase.md)
3. [`experiments.md`](experiments.md)
4. [`privacy.md`](privacy.md)
5. [`contributing.md`](contributing.md)

## What belongs in public docs

Public docs should explain:
- what the site is
- how the public codebase is organized
- what visitors can do on the site
- how to work on the code locally
- the site's public privacy stance

## What does not belong here

Public docs should not include:
- deployment internals
- infrastructure layout
- moderation tooling details
- admin route details
- sensitive operational procedures
- secrets or credential handling
- storage details that make abuse easier

Those belong in private docs.
