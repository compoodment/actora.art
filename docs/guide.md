# Guide

Start here if you want to understand the public side of actora.art.

## Reading order

1. [identity](identity.md) — what the site is trying to be
2. [codebase](codebase.md) — how the code is organized
3. [experiments](experiments.md) — what visitors can do
4. [privacy](privacy.md) — the site's privacy stance

## What belongs in public docs

Public docs explain what the project is, how the codebase is organized, and what visitors can experience. They describe capabilities and product intent, not how things are operated.

## What does not belong here

Deployment internals, infrastructure layout, moderation tooling, admin details, sensitive operational procedures, secrets handling, and anything that would make abuse easier. Those belong in private docs.
