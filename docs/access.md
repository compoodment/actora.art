# Access levels

actora.art has a few access levels so open public features can stay usable without turning every expensive or moderation-sensitive feature on for everyone.

## Guest

Guests can use the site without an account.

Guest access includes:
- the public pages and lab experiments
- Aurora chat in the current browser
- the Wall with normal public paint, erase, undo, and redo controls

Guest state is browser/cookie-scoped. It is useful for casual visits, but it is not meant to be a durable account identity.

## Signed-in user

Signed-in users use passkeys for account access.

Signed-in access adds:
- account identity across browsers/devices where the user signs in
- multiple saved Aurora chats, up to the normal account cap
- account-owned Wall activity and account Wall tool preferences
- saved Wall color slots

Most signed-in accounts use the normal public limits and the fast Aurora model.

## Trusted / friend+

Some known accounts may get trusted access, also called friend+ in project notes.

Trusted access can include:
- higher daily Aurora message allowance
- access to the smart Aurora model
- higher Wall paint and erase-refund allowances
- a higher saved-chat cap than normal users

This level exists for trusted testers and close users so heavier or more expensive features can be tried without opening them to drive-by abuse.

## Admin and owner

Admin and owner accounts are for operating the site.

At a public level:
- owner accounts are configured by the site owner
- admin-capable accounts can access private operational controls
- owner/admin details, private moderation tools, internal identifiers, storage layout, and operational procedures are not public documentation

If Aurora can answer a role question from this public page, it should. If someone asks for private admin operations, hidden state, secrets, or implementation details, Aurora should say that part is not public.
